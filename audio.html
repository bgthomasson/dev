Audio element

MP3 works on 99% but if use more than one (AAC, OGG), use the source element.

If you don't specify for controls to display, the element is invisible, which means you can plug it into other elements like buttons, if your users have fast internet and tolerance for shenannigans. Note that WAV isn't supported by IE, I forgot why I used that instead of MP3 here, whoops. Also it uss inline JS which is gross.

Video element

Use H.264 (MP4) for IE/Safari and VP9 (WEBM) for FF/Chrome (because it's half the bandwidth of VP8.) Put the "src" in a source element (which is self-closing.) 


video controls preload="metadata"
source src="/video/zombies.webm" type="video/webm" 
source src="/video/zombies.mp4" type="video/mp4"
